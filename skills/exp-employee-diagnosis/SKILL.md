# 员工体验诊断

分析员工 JTBD、服务目标与用户期待的匹配度，评估员工体验如何传导到客户体验。

## 触发条件

- 客户体验问题可能源于员工侧
- 员工能力、工具、权限、信息或情绪压力有问题
- 需要分析员工体验对客户体验的影响
- 需要改善员工支持和赋能

## 核心能力

### 1. 分析员工 JTBD

理解员工的工作任务和目标：
- 员工想要完成什么任务（功能任务）
- 员工想要感受到什么（情绪任务）
- 员工想要在同事/客户眼中呈现什么（社会任务）

### 2. 检查服务目标匹配度

评估员工的服务目标与用户期待是否匹配：
- 员工理解的用户目标是什么
- 员工实际交付的是什么
- 差距在哪里

### 3. 评估支持度

从以下维度评估员工是否得到足够支持：

| 支持类型 | 检查内容 |
|---------|---------|
| 信息支持 | 员工是否能方便获取用户信息、产品信息、政策信息 |
| 工具支持 | 员工使用的系统/工具是否好用、高效、稳定 |
| 知识支持 | 员工是否得到足够的培训、知识库、最佳实践 |
| 权限支持 | 员工是否有足够权限快速解决用户问题 |

### 4. 评估情绪压力

评估员工的情绪压力和倦怠风险：
- 工作量是否合理
- 是否有情绪疏导机制
- 是否感到被支持和赋能

### 5. 评估协作成本

评估跨部门协作是否顺畅：
- 员工是否需要协调多个部门才能解决用户问题
- 协作流程是否清晰高效

### 6. 分析传导机制

分析员工体验如何传导到客户体验：
- 员工情绪 → 服务态度 → 客户感受
- 员工工具效率 → 响应速度 → 客户满意度
- 员工信息获取 → 问题解决质量 → 客户信任

## 输出格式：员工体验分析报告

```
【员工体验分析报告】

▸ 员工任务：
  功能任务：[帮我...]
  情绪任务：[让我感到...]
  社会任务：[让我看起来...]

▸ JTBD：[员工的 JTBD]
▸ 服务目标：[员工理解的服务目标]
▸ 用户期待：[用户实际期待的是什么]

▸ 支持度评估：
  信息支持：[充足/不足]，问题：[...]
  工具支持：[充足/不足]，问题：[...]
  知识支持：[充足/不足]，问题：[...]
  权限支持：[充足/不足]，问题：[...]

▸ 情绪压力：[高/中/低]，问题：[...]
▸ 协作成本：[高/中/低]，问题：[...]

▸ 对客户体验的影响：
  [员工体验问题如何传导到客户体验]

▸ 改善机会：
  1. [...]
  2. [...]
  ...
```

## 使用方法

当客户体验问题可能源于员工侧时，调用本 skill 进行员工体验诊断，找出根本原并提供改善方向。

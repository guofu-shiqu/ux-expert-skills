# 触点审计

全面盘点用户与产品/品牌的所有接触触点，评估各触点体验质量，发现覆盖盲区和体验断点。

## 触发条件

- 需要全面盘点用户触点
- 需要评估各触点体验质量
- 需要发现触点覆盖盲区
- 需要优化跨触点体验一致性
- 需要设计或优化触点策略

## 核心能力

### 1. 触点发现与分类

从用户视角全面发现触点，按渠道和类型分类：

| 触点类型 | 典型触点 | 用户角色 | 体验重点 |
|---------|---------|---------|---------|
| 认知触点 | 广告、搜索结果、社媒、口碑 | 潜在用户 | 第一印象、价值传达 |
| 考虑触点 | 官网、App Store、评测、对比页 | 考虑用户 | 信息充分性、决策支持 |
| 获取触点 | 注册、下载、购买、开通 | 新用户 | 流畅度、信任感 |
| 使用触点 | 核心功能、搜索、设置、通知 | 活跃用户 | 效率、易用性 |
| 服务触点 | 帮助中心、客服、社区、更新日志 | 求助用户 | 响应速度、解决效率 |
| 离开触点 | 注销、取消订阅、投诉渠道 | 流失用户 | 退出顺畅性、挽回机会 |
| 传播触点 | 分享、评价、推荐、社媒提及 | 推荐者 | 分享动力、口碑管理 |

### 2. 触点审计维度

对每个触点从以下维度审计：

**体验质量维度**
- 易用性 — 用户能否轻松完成操作
- 可用性 — 功能是否稳定可用
- 一致性 — 跨触点体验是否连贯
- 情感性 — 触发的情绪是正还是负

**业务价值维度**
- 覆盖广度 — 触达了多少目标用户
- 转化效率 — 从进入触点到下一触点的转化率
- 用户满意度 — 该触点的 CSAT/NPS
- 品牌印记 — 用户记住了什么

**运营健康度维度**
- 数据可得性 — 该触点的数据是否可追踪
- 优化频率 — 该触点多久优化一次
- 责任人 — 该触点是否有明确负责团队
- 异常监测 — 是否有异常预警机制

### 3. 触点矩阵分析

以"体验重要性"和"当前体验质量"为两轴，将触点定位到矩阵：

```
高重要性 + 低质量 → 优先改善（短板触点）
高重要性 + 高质量 → 保持优势（核心触点）
低重要性 + 低质量 → 低优先级（可维持现状）
低重要性 + 高质量 → 过度投入（可考虑降成本）
```

### 4. 跨触点一致性检查

检查跨触点体验是否连贯：
- 视觉一致性 — 品牌视觉是否统一
- 语言一致性 — 文案语气是否一致
- 数据一致性 — 用户信息是否跨触点同步
- 流程一致性 — 跨触点流程是否顺畅衔接
- 情绪一致性 — 情绪曲线是否连贯

## 使用方法

当需要全面盘点和优化用户触点时调用本 skill。审计结果可与旅程分析、峰值设计、编排设计等 skill 衔接，将触点洞察转化为具体的体验优化动作。

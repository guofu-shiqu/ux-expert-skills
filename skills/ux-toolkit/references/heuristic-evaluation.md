# Nielsen 十大可用性启发式原则 — 检查清单

> 来源：Jakob Nielsen (1994, 更新于 2020)

## 十大原则

### 1. 系统状态可见性（Visibility of System Status）
- [ ] 用户操作后，系统是否在合理时间内给予反馈？
- [ ] 加载/处理过程中是否显示进度指示器？
- [ ] 当前所在位置/状态是否清晰可见（如面包屑导航）？

### 2. 系统与现实的匹配（Match between System and the Real World）
- [ ] 使用的语言和概念是否为用户熟悉的词汇？
- [ ] 信息排列是否遵循自然逻辑顺序？
- [ ] 图标和隐喻是否符合现实世界的认知？

### 3. 用户控制与自由（User Control and Freedom）
- [ ] 是否提供「撤销/重做」功能？
- [ ] 用户能否轻松退出当前操作（如返回按钮）？
- [ ] 是否有明确的「紧急出口」用于中断长流程？

### 4. 一致性与标准（Consistency and Standards）
- [ ] 相同功能在不同页面的交互方式是否一致？
- [ ] 是否符合平台设计规范（iOS HIG / Material Design）？
- [ ] 术语和按钮文案在产品内是否统一？

### 5. 错误预防（Error Prevention）
- [ ] 关键操作前是否提供确认？
- [ ] 输入框是否有格式限制和提示（如日期格式）？
- [ ] 是否有防止重复提交的机制？

### 6. 识别而非回忆（Recognition Rather than Recall）
- [ ] 选项和操作是否可见，而非依赖用户记忆？
- [ ] 表单是否提供默认值和自动补全？
- [ ] 帮助信息是否在需要时可见？

### 7. 灵活性与效率（Flexibility and Efficiency of Use）
- [ ] 是否支持快捷键/手势操作？
- [ ] 是否有个性化设置选项？
- [ ] 高级用户和新手是否各有合适的操作路径？

### 8. 美学与极简设计（Aesthetic and Minimalist Design）
- [ ] 每个页面是否只包含必要信息？
- [ ] 无关或低频信息是否隐藏/收起？
- [ ] 视觉层级是否引导用户关注核心内容？

### 9. 帮助用户识别、诊断和恢复错误（Help Users Recognize, Diagnose, and Recover from Errors）
- [ ] 错误提示是否使用通俗语言描述问题？
- [ ] 是否提供建设性的解决建议？
- [ ] 错误信息是否清晰指出问题所在位置？

### 10. 帮助与文档（Help and Documentation）
- [ ] 是否有易于搜索的帮助内容？
- [ ] 帮助内容是否按用户任务组织（而非按功能列表）？
- [ ] 帮助入口是否在需要时容易找到？

## 严重性评级

| 等级 | 说明 | 示例 |
|------|------|------|
| 0 | 无问题 | — |
| 1 | 表面问题（不影响任务完成） | 拼写错误、颜色不一致 |
| 2 | 轻微问题（造成困扰但可克服） | 按钮位置不直观 |
| 3 | 严重问题（导致任务失败或用户流失） | 关键流程无确认导致误操作 |
| 4 | 灾难性问题（必须立即修复） | 支付流程反复崩溃 |
